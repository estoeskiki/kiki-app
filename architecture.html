💳 Payment Flow (step by step)

  • 1. Customer taps "Pay" on kiosk
  • 2. App sends amount to bank terminal via ECR SDK
  • 3. Terminal prompts customer: insert / tap card
  • 4. Bank authorizes → terminal returns approved/declined
  • 5. On approval → app calls POST /orders to save order
  • 6. Supabase Realtime fires → KDS receives order instantly
  • 7. App calls printer SDK → receipt printed

📦 Supabase Data Model (core tables)

  • restaurants — id, name, location, owner_id
  • menus — id, restaurant_id, name, active
  • categories — id, menu_id, name, sort_order
  • items — id, category_id, name, price, image_url, available
  • orders — id, restaurant_id, status, total, created_at
  • order_items — id, order_id, item_id, qty, unit_price
  • kiosks — id, restaurant_id, serial, last_seen_at

📺 KDS — How It Works

  • Android tablet mounted in kitchen
  • Runs a simple React Native or web app
  • Subscribes to Supabase Realtime channel for its restaurant_id
  • New order appears instantly without polling
  • Kitchen staff taps "Done" → order status updates to completed
  • No extra backend needed — pure Supabase Realtime

🔧 Key Tech Decisions

  • React Native — shared codebase for kiosk + KDS apps
  • Supabase RLS — each restaurant only sees its own data
  • ECR via LAN — kiosk calls terminal on local network IP
  • MDM (Hexnode) — push app updates to all kiosks remotely
  • Offline fallback — queue orders locally if internet drops
  • Next.js API — handles business logic, payment confirmation
